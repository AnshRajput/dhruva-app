## 3. GitHub Repository Collision Check

**Primary Repository:** `AI4Bharat/Dhruva-Platform`  
**Status:** ⚠️ **SOFT COLLISION** (same ecosystem, different scope)

**Repository Details:**
- **URL:** https://github.com/AI4Bharat/Dhruva-Platform
- **Purpose:** Open-source platform for **serving language AI models at scale** (backend inference server)
- **Scope:** MLOps, model serving, API orchestration
- **Related repos:** Dhruva-Evaluation-Suite, Dhruva-Schema

**Related Repositories:**
1. **AI4Bharat/Dhruva-Evaluation-Suite** — Testing/benchmarking tool for Dhruva Platform
2. **AI4Bharat/Dhruva-Schema** — Data schema definitions

**Conflict Analysis:**
- **Scope Difference:** AI4Bharat's Dhruva = backend model serving platform. Proposed Dhruva = frontend Flutter app for model browsing & on-device inference.
- **Overlap:** Both are in the LLM ecosystem and use "Dhruva" as the brand.
- **Severity:** SOFT (different products, complementary functions, potential ecosystem collaboration)

**Risk Assessment:**
- GitHub's search may surface both projects when searching "Dhruva LLM"
- Users/contributors might assume connection (false positives)
- However, different repos, different organizations (AI4Bharat != personal/company)

**Recommendation:** 
1. **Proceed with caution** — "Dhruva" for the Flutter app is viable but acknowledge the AI4Bharat platform
2. **Consider ecosystem alignment:** Could the Flutter app specifically integrate/extend Dhruva-Platform? (Adds value, reduces confusion)
3. **If ecosystem integration is out of scope:** Add a note in README clarifying relationship: "Dhruva (Flutter app) is an independent on-device LLM browser; not affiliated with AI4Bharat/Dhruva-Platform."

**GitHub Status:** Safe to register repo as `dhruva-app` or `dhruva-flutter` to disambiguate from Dhruva-Platform
